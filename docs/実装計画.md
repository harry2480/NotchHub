# NotchHub 実装計画書

[`要件定義.md`](要件定義.md)（第24章 実装優先順位・第23章 MVP 成功条件）に基づく実装計画。

## 1. 実装フェーズ

### Phase 0: 基盤構築
- SPM 構成を作成し、最小デプロイメントターゲット macOS 14+ を実装済み。将来の機能方針は macOS 15+ を基準に検討する
- ディレクトリ構成（[`アーキテクチャ.md`](アーキテクチャ.md)）と Composition Root の骨組み
- 常駐化（メニューバー / `SMAppService` による Launch at Login）
- SwiftFormat / SwiftLint / CI（build + test）のセットアップ
- SQLite 接続基盤とマイグレーション機構（`~/Library/Application Support/NotchHub/`）

### Phase 1: ノッチ + ドラッグ UI（MVP コア）
- **Notch Core**: `NSPanel` によるノッチ／擬似ノッチウィンドウ、通常時の同化、展開・収縮アニメーション
- 展開トリガー（ドラッグ接近 / クリック / AI 承認待ち）と閉じる条件（ノッチ外クリック、補助 Esc）
- 最小状態の優先度制御（AI 承認待ち > ドラッグ中 > AirDrop/Share > メディア > 予定10分前）
- **Drag UI**: グローバルドラッグ検出、3 Zone（Shelf / Share / AirDrop）表示、Drop 時のみ実行、Dead Zone・強調・Toast・Undo
- 擬似ノッチ（ノッチなし Mac）サイズ・マルチディスプレイ対応

### Phase 2: Shelf + Search
- **Shelf**: BookmarkData による参照保持、テキスト/URL/Markdown の SQLite 保存、フォルダ保持
- 並び順（ピン留め → 新しい順）、上限（各 50 件）と超過時処理、寿命（永久/7日/30日/カスタム）
- 行操作（開く / Finder 表示 / 再ドラッグ / AirDrop / Share / ピン留め / 削除 / 全削除）
- 元ファイル削除時の自動除去
- **Search**: ファイル名・フォルダ名・URL・本文の全文検索

### Phase 3: 共有とスクリーンショット
- **AirDrop**: AirDrop Zone Drop → 共有 UI 起動、送信後動作（残す/削除）、履歴保存（送信先は保存しない）
- **Share Sheet**: `NSSharingServicePicker` 起動（Mail/Messages/Notes/Reminders、Discord/LINE は経由のみ）
- **Screenshot Auto Import**: ⌘+Shift+3/4/5 検出 → Shelf 自動追加 → Toast（ON/OFF）

### Phase 4: 補助機能
- **Claude Code Monitor**: hooks + local socket でセッション/状態/承認待ち検出、承認待ちでノッチ自動展開、Approve/Deny/Stop
- **Calendar**: EventKit で Next / Today 表示、過去予定非表示、クリックで Calendar.app 起動
- **Media**: AppleScript で Apple Music / Spotify の曲名・アーティスト・アートワーク表示と Play/Pause/Next/Previous
- **Codex CLI / Antigravity CLI Monitor**: Hooks + socket（Antigravity は Hooks > Plugins > SDK > Process の優先順）
- **Settings**: 各種設定項目、タブ構成（Shelf | Calendar | Media | AI）

## 2. 詳細実装メモ

### 2.1 ノッチウィンドウ（Platform / App）

```swift
// 非アクティブ・最前面・全 Space に表示するパネル
let panel = NSPanel(
    contentRect: rect,
    styleMask: [.borderless, .nonactivatingPanel],
    backing: .buffered,
    defer: false
)
panel.level = .statusBar              // 最前面
panel.collectionBehavior = [.canJoinAllSpaces, .stationary, .ignoresCycle]
panel.isOpaque = false
panel.backgroundColor = .clear
panel.contentView = NSHostingView(rootView: NotchRootView(viewModel: notchViewModel))
```

### 2.2 グローバルドラッグ検出（Platform）

- `NSEvent.addGlobalMonitorForEvents(matching: [.leftMouseDragged])` でドラッグ位置を監視
- カーソルが画面上部中央へ近づいたらノッチ展開・3 Zone 表示
- カーソルが存在するディスプレイのノッチを対象にする（マルチディスプレイ）
- ドロップ受けは `NSPanel` 側の `NSDraggingDestination` / SwiftUI `onDrop`

### 2.3 AI CLI 連携（Platform / AIMonitor）

- ローカル `Unix Domain Socket` を待ち受け、各 CLI の hooks からイベント受信
- イベント（SessionStart / PreToolUse / PermissionRequest / PostToolUse / Stop 等）を AIMonitor の状態へ反映
- PermissionRequest 受信時にノッチ自動展開し、Approve/Deny を hook 応答として返す

### 2.4 Shelf 参照保持（Repository / Platform）

- 追加時に Security Scoped Bookmark を生成し SQLite に保存
- アクセス時 `startAccessingSecurityScopedResource()` を対で使用
- 共有時のみ一時ファイル（.txt / .md / .webloc）を `cache/` に生成
