# NotchHub インフラ規約

Xcode / Swift によるビルド・署名・配布の構成と規約。Web サーバーやクラウド DB は持たず、すべてローカル（ユーザーの Mac）で完結するスタンドアロンの macOS アプリである。

---

## プロジェクト構成

```text
NotchHub/
├── NotchHub.xcodeproj      # （または Package.swift によるSPM構成）
├── Sources/NotchHub/       # アプリ本体（アーキテクチャ.md 参照）
├── Tests/                  # XCTest / Swift Testing
├── Resources/              # アセット・Info.plist・entitlements
└── docs/                   # 設計ドキュメント
```

- 機能モジュールは将来的にローカル Swift Package（ターゲット）へ分割し、レイヤー境界をコンパイル時に強制することを推奨

---

## ローカルデータ配置

クラウドを使わず、ユーザーローカルに永続化する。

```text
~/Library/Application Support/NotchHub/
├── shelf.db             # Shelf アイテム（参照保持・テキスト本文）
├── settings.db          # 設定
├── airdrop_history.db   # AirDrop 履歴
├── cache/               # 一時ファイル（共有時の .txt/.md/.webloc 等）
└── thumbnails/          # サムネイルキャッシュ
```

- DB は SQLite。スキーマ変更はバージョン管理されたマイグレーションで適用する（[`リポジトリ層設計規約.md`](リポジトリ層設計規約.md) 参照）
- ファイルは原本をコピーせず BookmarkData（Security Scoped Bookmark）で参照保持する

## ビルド・配布

| 環境 | トリガー | 手段 | 備考 |
|---|---|---|---|
| Debug | ローカル開発 | `swift build` / `swift run` | サンドボックス・権限はローカルで検証 |
| Release | タグ付け / リリース手順 | Release build → `.app` 化 → 署名 → Notarization | 配布時のみ Developer ID 署名 + Apple 公証（notarytool）→ DMG / zip 配布 |

### ビルド設定

- 配布形態: Developer ID 署名済み `.app`（Mac App Store 外配布を想定）
- 署名: Developer ID Application 証明書 + `notarytool` による公証 + `stapler` でチケット添付
- 最小デプロイメントターゲット: macOS 14+
- アーキテクチャ: Universal（Apple Silicon + Intel）

### 主なコマンド

```sh
# 品質チェック一括
make verify

# ビルド
swift build

# テスト
./scripts/swift-test.sh

# リリースビルド（配布時はこの後に .app 化・署名・公証を行う）
swift build -c release
```

## 権限・entitlements

NotchHub は OS の機微 API を多用するため、entitlements と TCC（プライバシー）権限を正しく設定する。

| 権限 | 用途 |
|---|---|
| Files & Folders | Shelf のファイル参照保持・読み取り |
| Calendar | EventKit による予定取得 |
| Accessibility | グローバルドラッグ検出・ノッチ制御 |
| Notifications | Toast / 承認待ち通知 |
| Automation | AppleScript による Apple Music / Spotify 制御 |

- **方針: 権限は必要になった時点で要求する。初回に一括要求しない**
- App Sandbox を有効にする場合、Security Scoped Bookmark によるファイル再アクセスと、必要な temporary-exception の整合を取ること
- Launch at Login は `SMAppService`（Launch Services）で実装する

## サードパーティ依存

- 依存は最小限に保つ。標準フレームワーク（SwiftUI / AppKit / EventKit / Security 等）を優先する
- 外部ライブラリを追加する場合は SwiftPM で管理し、ライセンスを同梱する（GPL コードは直接流用しない。MIT/Apache はライセンス同梱）
- 参考リポジトリ（NotchDrop / Atoll / Notchy / Vibe Notch）の扱いは [`要件定義.md`](要件定義.md) 第22章を参照
