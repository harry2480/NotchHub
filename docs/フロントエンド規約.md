# NotchHub UI（SwiftUI）実装規約

SwiftUI + AppKit によるノッチ常駐 UI の実装規約。

> 「フロントエンド」はネイティブ macOS アプリの **UI 層（View + ViewModel）** を指す。

---

## ディレクトリ構成

`Sources/NotchHub/` 配下:

| ディレクトリ | 責務 |
|---|---|
| `Features/*/Views/` | SwiftUI ビュー。表示と入力のみ |
| `Features/*/ViewModels/` | 画面状態（`@MainActor @Observable`）。Service を呼ぶ |
| `App/` | エントリ・メニューバー・ノッチウィンドウ（AppKit）・Composition Root |
| `Core/` | 共有コンポーネント・モデル・ユーティリティ |
| `Services/` / `Repositories/` / `Platform/` | サーバーサイド相当（[`アーキテクチャ.md`](アーキテクチャ.md) 参照） |

---

## SwiftUI / AppKit の使い分け

- **コンテンツ描画は SwiftUI をデフォルトとする。** ノッチ展開時のタブ・リスト・操作 UI はすべて SwiftUI
- **AppKit を使うのは以下に限定する**:
  - ノッチ／擬似ノッチのウィンドウ（`NSPanel`：非アクティブ・最前面・全 Space 表示）
  - グローバルなドラッグ／イベント監視（`NSEvent` global monitor）
  - ドラッグ&ドロップの低レベル制御（`NSDraggingDestination` 等、SwiftUI の `onDrop` で不足する場合）
- **AppKit 実装は `App/` および `Platform/` に隔離し、Feature の View からは SwiftUI として扱えるようにする**（`NSViewRepresentable` / `NSHostingView` で橋渡し）

---

## 状態管理とデータフロー

- **状態は ViewModel に集約する。** `@MainActor @Observable`（または `ObservableObject`）で公開
- **View 内 `@State` は純粋にローカルな UI 状態のみ**（アニメーションフラグ等）。ドメイン状態を持たせない
- **データ取得・更新は必ず Service 経由。** View / ViewModel から DB・OS API を直接呼ばない
- **副作用（AirDrop 起動・Share Sheet 起動・ファイル削除）は Service のメソッドとして実行し、結果を ViewModel が受けて状態反映する**
- **横断状態（ノッチ展開状態・現在タブ・承認待ち件数）は共有 ViewModel / AppState として Composition Root から注入する**

---

## ノッチ UI 固有規約

- 展開トリガーは「ドラッグ接近 / クリック / AI 承認待ち」のみ。Hover・接近で展開しない
- 閉じる主導線はノッチ外クリック。Esc は補助
- 最小状態は優先度順に **1 状態のみ** 表示
- サイズ・余白は定数（[`スタイルガイド.md`](スタイルガイド.md) の `NotchLayout` 等）から参照し、直値を書かない
- 外部ディスプレイ対応: ドラッグ中はカーソルが存在するディスプレイのノッチを展開する（複数スクリーンを ViewModel が把握する）
