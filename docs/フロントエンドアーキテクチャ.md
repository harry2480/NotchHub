## 6. 実装ガイドライン

### コーディング規約
1. **コンポーネント設計**
   - 単一責任の原則
   - Props Interface の明確な定義
   - コンポーネントの適切な分割
2. **型定義**
   - 全てのコンポーネントで TypeScript を使用
   - any 型の使用禁止
   - 適切な型の共有と再利用
3. **スタイリング**
   - {{STYLING_RULE_1 / 例：Tailwind CSS のユーティリティファースト}}
   - {{STYLING_RULE_2 / 例：カスタムコンポーネントでの cn()ヘルパー使用}}
   - レスポンシブデザインの徹底

### アーキテクチャ原則
1. **Feature-based 構造**: 機能ごとにモジュール化
2. **明確な責任分離**: API、Components、Hooks、Types の分離
3. **依存関係の管理**: 循環参照を避ける
4. **テスタビリティ**: テストしやすいコード設計
