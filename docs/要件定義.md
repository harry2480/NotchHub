# NotchHub 要件定義書 v1.0

## 1. プロダクト概要

NotchHub は、Mac のノッチを活用した「共有」と「一時保管」のための常駐アプリケーションである。

### 1.1 主役機能

| 機能 | 概要 |
|------|------|
| AirDrop 高速化 | ノッチへドラッグするだけで AirDrop 共有 UI を即起動する |
| Shelf（一時保管） | ファイル・フォルダ・テキスト類を一時的に保管する |

### 1.2 補助機能

- macOS Share Sheet
- Claude Code Monitor
- OpenAI Codex CLI Monitor
- Antigravity CLI Monitor
- Calendar 表示
- Media Control

---

## 2. プロダクトの位置付け

NotchHub の中核は **「AirDrop + Shelf」** であり、AI CLI Monitor・Calendar・Media は補助機能と位置付ける。

本プロダクトは「AI アシスタント管理アプリ」ではなく、**「ノッチを活用した共有・一時保管ハブ」** として定義する。

```text
主役: AirDrop + Shelf
補助: AI CLI Monitor / Calendar / Media
```

## 5. ノッチ UI

### 5.1 通常状態

```text
[ black notch ]
```

- 通常時は目立たせない
- 余計な情報は表示しない
- クリック可能領域としてのみ存在する

### 5.2 展開条件

**展開する:**
- ドラッグ中に画面上部中央へ近づいた時
- ノッチをクリックした時
- AI 承認待ちが発生した時

**展開しない:**
- Hover のみ
- カーソル接近のみ
- 常時表示

### 5.3 閉じる条件

- 主動線: ノッチ外をクリック
- 補助: Esc キー対応可（主動線にはしない）

---

## 6. ノッチ最小状態

### 6.1 基本方針

通常時は何も表示せず、ノッチに同化する。

### 6.2 状態表示の優先順位

1. AI 承認待ち
2. ドラッグ中
3. AirDrop / Share 実行中
4. メディア再生中
5. 次の予定 10 分前

### 6.3 表示例

```text
通常        : [ black notch ]
AI 承認待ち : [ ! ]
メディア再生: [ ⏯ ]
予定10分前  : [ 10m ]
AirDrop     : [ Sent ]
```

## 7. ドラッグ UI

### 7.1 フロー

```text
Drag Start → 画面上部中央へ移動 → ノッチ展開 → 3 Zone 表示
→ Zone 選択 → Drop → 処理実行
```

### 7.2 レイアウト

```text
┌─────────────────────────────────────┐
│ Shelf           Share       AirDrop │
│ Keep            Choose      Send    │
└─────────────────────────────────────┘
```

### 7.3 Drop Zone

| 位置 | Zone | 動作 | 保存 |
|------|------|------|------|
| 左 | Shelf | Shelf へ追加（Share / AirDrop は実行しない） | Shelf へ保存 |
| 中央 | Share | macOS Share Sheet 起動 | Shelf へ保存しない |
| 右 | AirDrop | AirDrop 起動 | 設定による |

### 7.4 誤操作防止

- Hover 発火禁止
- 接近発火禁止
- Drop 時のみ実行
- Dead Zone 導入
- Drop 前に強調表示
- Toast 表示
- Undo 対応

---

## 8. Shelf

### 8.1 目的

「あとで使う」「あとで送る」「一時保管する」を実現する。

### 8.2 対象

- ファイル
- フォルダ
- テキスト
- URL
- Markdown
- 画像
- スクリーンショット

### 8.3 ファイル保存方式

| 項目 | 内容 |
|------|------|
| 方式 | 参照保持 |
| 保存内容 | BookmarkData / メタデータ |
| コピー | しない |

元ファイルが削除された場合は、Shelf から自動的に削除する。

### 8.4 フォルダ

- フォルダごと保持する
- zip 化しない
- 展開表示しない

### 8.5 テキスト

- 保存先: SQLite
- 保存対象: テキスト / URL / Markdown

#### 一時ファイル化（共有時のみ生成）

| 種類 | 拡張子 |
|------|--------|
| Text | .txt |
| Markdown | .md |
| URL | .webloc |

### 8.6 DB

保存先:

```text
~/Library/Application Support/NotchHub/
```

内容:

```text
shelf.db
settings.db
airdrop_history.db
cache/
thumbnails/
```

### 8.7 並び順

```text
ピン留め → 新しい順
```

### 8.8 表示方式

```text
📌 proposal.pdf
📁 project-folder
🖼 screenshot.png
🔗 github.com
📝 memo
```

表示要素: アイコン / 名前

### 8.9 Shelf 寿命（設定）

- 永久保存
- 7 日
- 30 日
- カスタム

### 8.10 Shelf 上限

| 区分 | 上限 |
|------|------|
| 通常 | 50 件 |
| ピン留め | 50 件 |

**上限超過時:**
- 通常: 最古の未ピン留めから削除
- ピン留め: 警告のみ

### 8.11 操作

開く / Finder 表示 / 再ドラッグ / AirDrop / Share / ピン留め / 削除 / 全削除 / 検索

### 8.12 検索（MVP 必須）

検索対象: ファイル名 / フォルダ名 / URL / テキスト本文 / Markdown 本文

### 8.13 タグ

MVP 対象外。

## 10. AirDrop

### 10.1 起動

```text
AirDrop Zone へ Drop → AirDrop 共有 UI 起動
```

### 10.2 対象

- ファイル
- フォルダ
- 複数ファイル
- Shelf アイテム

### 10.3 既定動作

送信後は Shelf へ残す。

### 10.4 設定

- 残す
- 削除する

### 10.5 AirDrop 履歴

**保存する:** 名前 / 種類 / 日時 / 元パス / 結果

**保存しない:** 送信先
